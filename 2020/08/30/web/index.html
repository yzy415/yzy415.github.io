布局大多是基于 box model的，每一个在在页面中占据一定空间的盒子都有padding，border, margin这三个属性。

  • padding, the space around the content. In the example below, it is the space around the paragraph text.
  • border, the solid line that is just outside the padding.
  • margin, the space around the outside of the border.

The body is a block element, meaning it takes up space on the page. A block element can have margin and other spacing values applied to it. In contrast, images are inline elements. It is not possible to apply margin or spacing values to inline elements. So to apply margins to the image, we must give the image block-level behavior using display: block;

Selector

Selector 内部有空格的，选择的是在前者其中的后者，Selector内部有加号的，选择的是紧跟着前者的后者。

差之毫厘，失之千里

  • p.special说的是类名为special的段落，p .special说的是在段落内类名为special的元素。
  • article:first-child说的作为其他任意元素的第一个孩子的article，article :first-child说的是article内的所有子元素的第一个孩子。可以使用article *:first-child提高可读性，也表示article内的所有子元素的第一个孩子。
  • .notebox.warning适用于class="notebox warning"，而.notebox, .warning只是一个list，其specificity比前者低，.notebox ,.warning只查找在.notebox里的.warning。
  • h1#heading适用于<h1 id="heading">

Cascade, specificity and Ineheritance

Cascade，相同specificity规则适用于同一个元素时，先来后到。

Specificty，不同specificity的规则适用于同一个元素时，指令更加明确（having more specificity）的Rule会覆盖不那么明确的指令，如类选择器比元素选择器会更加的明确。（p.box也是元素选择器而不是类选择器）(元素选择器和类选择器是我自己造的词)

Inheritance指的是有一些属性的值会默认继承他的父元素的属性值的行为。

Specificity

使用四位数来指明选择器的明确度

  1. Thousands: Score one in this column if the declaration is inside a style attribute, aka inline styles. Such declarations don’t have selectors, so their specificity is always simply 1000.
  2. Hundreds: Score one in this column for each ID selector contained inside the overall selector.
  3. Tens: Score one in this column for each class selector, attribute selector, or pseudo-class contained inside the overall selector.
  4. Ones: Score one in this column for each element selector or pseudo-element contained inside the overall selector.

NOTE:没有人说过这是十进制的四位数，实际上后一位的数字不管有多大其明确度都要比前一位的数字小

Wrap up

有些属性值可以自动继承，明确声明的属性值会覆盖掉继承的。可以使用inherit, initial, unset来控制属性值的继承。

明确度相同看前后位置，不同CSS的前后位置如下：

  1. Declarations in user agent style sheets (e.g. the browser’s default styles, used when no other styling is set).
  2. Normal declarations in user style sheets (custom styles set by a user).
  3. Normal declarations in author style sheets (these are the styles set by us, the web developers).（开发人员的样式可以覆盖用户的，以保持开发者的本意）
  4. Important declarations in author style sheets
  5. Important declarations in user style sheets

The Box Model

两种盒子

CSS有两种盒子：block boxes和inline boxes，block boxes拥有盒子模型的所有特性，而inline box只有部分行为遵循盒子模型。

更改display的值会变更盒子的类型，进而影响其和其他元素之间的排列在布局中的排列方式。display:flex影响的是内部的布局。

  • Content box: The area where your content is displayed, which can be sized using properties like width and height.
  • Padding box: The padding sits around the content as white space; its size can be controlled using padding and related properties.
  • Border box: The border box wraps the content and any padding. Its size and style can be controlled using border and related properties.
  • Margin box: The margin is the outermost layer, wrapping the content, padding and border as whitespace between this box and other elements. Its size can be controlled using margin and related properties.（padding和margin都是whitespace，而border是可以控制样式的，content可以控制长宽）

<span>作为内联盒子，无法设置其宽和高，可以设置border, marigin和padding，但无法让其他的内容挪位置。

两个模型

标准模型中：给盒子赋予的宽和高，是给content box的，padding，boder，margin都是后加的。width，height，padding, boder, marigin共同确定了一个盒子占据的空间。margin没有被算入盒子的实际的大小，虽然它影响了盒子占据的空间大小，但影响的只是盒子的外部占据空间的大小，盒子的实际大小只有content, padding, border三者决定。

浏览器加载文件流程的简化版本

  1. The browser loads the HTML (e.g. receives it from the network).

  2. It converts the HTML into a DOM (Document Object Model). The DOM represents the document in the computer’s memory. The DOM is explained in a bit more detail in the next section.

  3. The browser then fetches most of the resources that are linked to by the HTML document, such as embedded images and videos … and linked CSS! JavaScript is handled a bit later on in the process, and we won’t talk about it here to keep things simpler.

  4. The browser parses the fetched CSS, and sorts the different rules by their selector types into different “buckets”, e.g. element, class, ID, and so on. Based on the selectors it finds, it works out which rules should be applied to which nodes in the DOM, and attaches style to them as required (this intermediate step is called a render tree).

  5. The render tree is laid out in the structure it should appear in after the rules have been applied to it.

  6. The visual display of the page is shown on the screen (this stage is called painting).

Some properties like font, background, padding, border, and margin are called shorthand properties. This is because shorthand properties set several values in a single line.

CSS属性的初始值是其默认值，如其定义表中所列。 初始值的使用取决于属性是否被继承：

  • 对于继承属性，初始值只能被用于没有指定值的根元素上。
  • 对于非继承属性，初始值可以被用于任意没有指定值的元素上。
